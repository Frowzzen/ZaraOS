You are helping me build the first prototype of a project called ZaraOS.

IMPORTANT CONTEXT:
ZaraOS is not a traditional Linux distro project yet. The first version is an AI-native desktop assistant and operating environment that will later be packaged into a custom bootable Linux-based USB operating system.

The goal is to build the main ZaraOS software layer first as a cross-platform desktop application. Later, this app will be installed into a customized Linux ISO using tools like Cubic, then flashed to a USB drive for testing.

ZaraOS should feel like a futuristic AI operating system powered by a main assistant named Zara, similar in concept to Jarvis.

CORE PRODUCT VISION:
ZaraOS is a local-first, privacy-focused, AI-native operating environment where voice and hand gestures are the primary controls. Keyboard and mouse are secondary fallback controls.

The system should be designed for:
- USB bootable Linux-based deployment later
- high performance on modest hardware
- broad hardware compatibility
- local-first AI processing
- optional user-provided API keys for external AI engines
- strong privacy and security
- future third-party app/plugin development

VERY IMPORTANT COMPATIBILITY NOTE:
Design the application layer to run on Linux first, but keep the codebase portable so it can later support Windows and macOS. The USB bootable OS itself will most realistically target standard x86_64 PCs first. True universal USB booting across Windows laptops, Linux PCs, Intel Macs, and Apple Silicon MacBooks may require separate builds or different boot strategies later. Do not pretend one ISO will automatically boot every Mac and every PC. Build the app in a portable way so we can support more hardware over time.
