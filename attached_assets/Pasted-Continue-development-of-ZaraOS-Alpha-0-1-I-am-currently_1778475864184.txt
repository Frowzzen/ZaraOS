PROJECT CONTEXT:
ZaraOS is a local-first, privacy-focused, AI-native operating environment. It will eventually be packaged into a bootable Linux-based USB operating system, but right now we are building the software layer first.

The product vision is:
- Voice-first
- Gesture-first
- Keyboard and mouse as secondary fallback
- Local AI by default
- Cloud AI only if the user brings their own API key
- No company-paid API calls
- No unnecessary paid services
- No forced cloud backend
- Lightweight, secure, modular, and ready for Tauri desktop packaging later

TASK 2 — Create A Central Zara Runtime

Create a central runtime module, something like:

src/core/zara-runtime.ts

It should expose methods like:

- initialize()
- executeCommand(input: string, source: "voice" | "gesture" | "keyboard" | "system")
- requestPermission(permission)
- getSystemStatus()
- getPrivacyStatus()
- selectAIProvider(provider)
- sendAssistantMessage(message)
- launchApp(appId)
- registerPlugin(pluginManifest)

The runtime should become the brain of ZaraOS.

All commands from:
- Zara Assistant
- Zara Console
- Voice Engine
- Gesture Engine
- App Launcher

should eventually flow through this runtime.

TASK 4 — Improve Privacy And Permissions

Create a basic permissions system.

Permission categories:
- microphone
- camera
- local_ai
- cloud_ai
- network
- files
- system_actions
- plugins
- developer_mode

Rules:
- Mic and camera require explicit user toggle.
- Cloud AI is disabled by default.
- Local AI is enabled by default.
- System actions require permission.
- Destructive actions require confirmation.
- Plugins must declare permissions.
- API keys must never be logged.
- API keys must never be shown unmasked.
- API keys must never be sent anywhere in Alpha 0.1.

Add a permissions utility module if needed.
