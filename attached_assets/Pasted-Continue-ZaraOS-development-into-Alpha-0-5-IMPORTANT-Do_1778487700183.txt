TASK 1 — Secure Storage Abstraction

Create a secure storage abstraction layer.

Suggested location:
src/core/security/secure-storage.ts

Goals:
- Replace direct API key localStorage access with a storage interface.
- For browser Alpha 0.5, use Web Crypto AES-GCM encryption where possible.
- Store encrypted values in IndexedDB if practical.
- If IndexedDB is too much for this pass, create the abstraction and keep a clearly labeled fallback.
- Never log raw keys.
- Never return raw keys to UI unless the user is actively editing.
- Always mask keys in display state.

Create docs:
docs/SECURE_STORAGE_MODEL.md

Explain:
- Alpha browser limitations
- why localStorage is not secure enough long-term
- encrypted IndexedDB plan
- future Tauri OS keychain plan
- future Linux secret service / keyring plan
